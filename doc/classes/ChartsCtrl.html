    Class ChartsCtrl

    Class defined in: app/js/controllers/ChartsCtrl.js:7

    A controller that serves the container of DC.js and Canvas.js charts in the index.html view.
    It uses two custom angular services:

    • DCChartsService: generates the DC.js charts
    • CanvasChartsService: generates the Canvas.js charts

    It defines three methods that should be explained:

    • renderAll(): a method that launches automatically when all the DOM elements needed for rendering all the charts from the $rootScope.droppedCharts list where generated (but charts not yet traced). This is done by using angular $last directive. When the method is launched, if the boolean $scope.reload is true, it traces all the charts from the $rootScope.droppedCharts list. It is done this way, because in order to trace a DC.js or Canvas.js charts, all the DOM containers (divs) should be already present (and we are generating them progrmatically).
    • onDropChart(): a method that traces only one chart, when the 'movable chart' is dropped to the charts container. It also adds it to the $rootScope.droppedCharts list, which will trigger automatically the renderAll() function, that will retrace all for the second time. To prevent this, the onDropChart() method sets the variable $scope.reload to false.
    • reloadCharts(): a method that deletes all the DOM elements chart containers, and then reintroduces them. Which triggers the renderAll() method that traces all the charts. To allow it, the reloadCharts() sets the variable $scope.reload to true.
